transaction 
 : 데이터베이스의 상태를 변화시키기 위해 수행하는 작업 단위
 : commit & rollback을 하기 전까지의 과정

성질
 - 원자성 : 모든 명렁을 완벽히 수행, 전부가 완벽히 수행되지 않고 오류가 나면 전부 취소
 - 일관성 : 실행을 성공하면 일관성 있는 데이터베이스 상태로 변환, 
            수행 전과 수행 후의 상태가 같아야 함
 - 독립성 : 둘 이상의 트랜잭션이 싱행되는 경우 순차적 진행
            (다른 트랜잭션의 연산이 끼어들 수 없음)
 - 영속성 : 완료된 트랜잭션의 결과는 영구적으로 데이터베이스에 반영이 되어야 함

상태
Active(활동) - failed(실패) - aborted(철회)
             - partially commited(부분 완료) - commited(완료)
