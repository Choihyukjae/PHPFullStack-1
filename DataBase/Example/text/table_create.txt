CREATE : 

제약조건 : 잘못된 값을 입력하는 것을 방지
* alter로 수정 가능 
(alter table 테이블명 modify column 컬럼명 데이터타입 null/not null)

primary key 
 - 테이블 생성 시 고유 한 개의 pk 설정 (= 주민번호/테이블에서 대표되는 키&기본키)
 - 중복, null 불가
 - 고유 인덱스 자동 생성
 - 테이블의 데이터를 쉽고 빠르게 찾도록 도와줌

foreign key 
 - 테이블을 다른 테이블과 연결 (= 기준이 되는 테이블의 내용을 참조해서 레코드 입력)
 - 테이블 간 잘못된 매핑 방지
 - FK를 선언한 테이블이 하위 테이블 
 - 참초하는 컬럼과 데이터 타입이 일치해야 함 (참초 테이블(1,2,3) = 하위테이블(1,2,3))
 - 참조되는 테이블의 필드에 PK(primary Key), Unique 제약 조건이 설정되어 있어야 함

Unique 
 - 고유키
 - 중복 값 불가, 여러 개의 null 허용
 - 고유 인덱스 자동 생성
 - 무결성 체크에 사용

pk / unique 차이점
null X / null O 
한 테이블 당 하나의 필드에만 설정 / 한 테이블의 여러 필드에 설정

null(공백 가능), not null(공백 불가능)
 - 해당 컬럼의 null 가능 여부

default 
 - 컬럼의 기본값 설정
 - 레코드에 해당 필드 값을 전달하지 않으면 자동으로 설정된 기본 값 저장

check 
 - 데이터의 점검을 하는 제약 조건
